How This
Site Works.

A plain-English guide to every file in this project — written for someone who has never written code before.

How to view the site in your browser

  1. 1
    Open File Explorer and navigate to the Ataul Site folder on your computer.
  2. 2
    Double-click the file called index.html. It will open automatically in your default web browser (Chrome, Edge, Firefox, etc.).
  3. 3
    To navigate between pages, use the links in the nav bar just like a real website. Each page (about.html, gallery.html, contact.html) can also be opened directly by double-clicking it.
  4. 4
    Every time you save a change to a file, go back to the browser and press F5 (or Ctrl + R) to refresh and see the update.
💡 No internet required. Because all the images and styles are now local files inside the project folder, the site works completely offline. The only things that need internet are the Google Fonts (the typography) and the Instagram link.

Core concept

The three languages

Every website is built from three languages working together. Think of building a magazine layout:

Language Role Design analogy
HTML Structure & content The raw text and images placed on a page — no styling at all
CSS Visual design Font choices, colors, spacing, sizing — how everything looks
JavaScript Interactive behavior Buttons that respond, animations, things that change when you click or scroll
How they connect Each HTML file links to style.css and script.js at the top and bottom of the file. That is why you only need one CSS and one JS file for the whole site — every page shares them.

Understanding HTML

HTML uses tags — labels wrapped in angle brackets — to describe what each piece of content is.

How tags work

Every tag has an opening version and a closing version (with a /). The content sits between them.

<h1>Hello World</h1>

Tags can also carry attributes — extra settings inside the opening tag:

<a href="about.html" class="active">About</a>

Understanding CSS

CSS rules follow a simple pattern: who to target, then what to do to them.

.hero-name {
  font-size: 200px;
  color: white;
}
/* .hero-name = target anything with class="hero-name"
   Everything inside {} is applied to it */

Color variables (lines 7–26)

Instead of writing a color code like #c8ff3e everywhere, the site uses named variables. Change one line, and the color updates everywhere on the site.

:root {
  --bg:      #0f0f10;   /* main background — near black */
  --ink:     #f5f5f3;   /* main text — near white */
  --lime:    #c8ff3e;   /* accent color — bright lime green */
  --muted:   #a8a8a3;   /* secondary text — grey */
}

/* When JS adds class="light" to the body,
   these values override the dark ones above: */
body.light {
  --bg:   #f4f4f1;   /* now a light cream */
  --ink:  #0f0f10;   /* now dark text */
}

The CSS Reset (lines 29–34)

Browsers have their own default spacing. This removes all of it so you start from zero.

* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
  /* * means "every single element on the page" */
}

The Navbar — floating pill (lines 60–76)

.navbar {
  position: fixed;          /* stays on screen even when you scroll */
  top: 18px;
  left: 50%;                 /* move left edge to center */
  transform: translateX(-50%); /* shift back left by half its width → perfectly centered */
  border-radius: 999px;     /* a huge number = pill shape */
  z-index: 100;             /* layer number — 100 keeps it always on top */
  transition: all 0.3s ease; /* any change animates smoothly over 0.3 seconds */
}

Responsive font sizes (clamp)

The big name on the hero scales automatically with screen size — never too big, never too small.

.hero-name {
  font-size: clamp(64px, 14vw, 200px);
  /* clamp(minimum, preferred, maximum)
     14vw = 14% of the viewport width
     On a phone it might be 70px. On a huge monitor it might be 200px. -->
}

mix-blend-mode: difference

This is a CSS version of Photoshop's blending modes. The name text inverts against whatever is behind it — light where the background is dark, dark where the background is light. The name is always readable.

Animations with @keyframes

@keyframes bounce {
  0%, 100% { transform: translateY(0);   }  /* start and end: normal position */
      50%  { transform: translateY(8px); }  /* midpoint: 8px lower */
}
/* Applied to the "scroll ↓" hint for a gentle looping bounce */

The marquee scrolling strip

@keyframes scroll {
  from { transform: translateX(0); }
  to   { transform: translateX(-50%); }
}
/* Slides the track left by exactly half its width.
   Since the content is duplicated, it loops seamlessly. */

The masonry (Pinterest) gallery

.masonry {
  column-count: 3;   /* split into 3 columns */
  column-gap: 16px;
}
.m-item {
  break-inside: avoid; /* never split one image across two columns */
}

Fade-in on scroll — two-state system

.fade-in {
  opacity: 0;                  /* invisible by default */
  transform: translateY(24px); /* shifted 24px down by default */
  transition: opacity 0.8s ease, transform 0.8s ease; /* animate any change smoothly */
}
.fade-in.visible {
  opacity: 1;                  /* fully visible */
  transform: translateY(0);   /* back to normal position */
}
/* JavaScript adds class "visible" when element scrolls into view → CSS transition fires */

@media (max-width: 760px) {

Understanding JavaScript

JavaScript reacts to things — clicks, scrolling, time — and changes the page in response. Unlike HTML and CSS which just describe how the page looks, JS does things.

The core pattern "Listen to this element. When something happens, run this code." That is almost everything JavaScript does on this site.

1 — Dark / Light mode toggle

// Find the button in the HTML by its id
const themeBtn = document.getElementById('themeBtn');

// When the page loads, check if the user previously chose light mode
if (localStorage.getItem('theme') === 'light') {
  document.body.classList.add('light');
}
// localStorage = a tiny storage area in the browser that remembers
//               settings even after the tab is closed

// When the button is clicked...
themeBtn.addEventListener('click', function() {
  document.body.classList.toggle('light');
  // .toggle = if the class is missing, add it. If it's there, remove it.

  if (document.body.classList.contains('light')) {
    localStorage.setItem('theme', 'light');  // save the choice
  } else {
    localStorage.setItem('theme', 'dark');
  }
});

2 — Hamburger menu (mobile)

hamburger.addEventListener('click', function() {
  navLinks.classList.toggle('open');
  // CSS: .nav-links { display:none }  →  hidden
  // CSS: .nav-links.open { display:flex } →  visible
  // Toggling the class "open" switches between those two states
});

// Also close the menu when any nav link is clicked
for (let i = 0; i < links.length; i++) {
  links[i].addEventListener('click', function() {
    navLinks.classList.remove('open');
  });
}

3 — Fade-in on scroll

// IntersectionObserver watches whether elements are visible in the viewport
const observer = new IntersectionObserver(function(entries) {
  entries.forEach(function(entry) {
    if (entry.isIntersecting) {           // "is this element on screen?"
      entry.target.classList.add('visible'); // yes → add "visible" → CSS fades it in
    }
  });
}, { threshold: 0.15 });  // trigger when 15% of the element is visible

fadeElements.forEach(function(el) {
  observer.observe(el);  // watch each fade-in element
});

5 — Navbar shrinks on scroll

window.addEventListener('scroll', function() {
  if (window.scrollY > 30) {    // scrolled more than 30px?
    navbar.style.top = '10px';  // move navbar a bit closer to top edge

How to replace placeholder images with real photos

  1. Export or save your real photo as a .jpg or .png file (e.g. my-car.jpg).
  2. Copy that file into the images/ folder inside the Ataul Site folder.
  3. Open the relevant HTML file (e.g. gallery.html) in a text editor.
  4. Find the line that says src="images/gallery-01-car.svg" and change it to src="images/my-car.jpg". Do not change anything else on that line.
  5. Save the file, go to your browser, and press F5 to refresh.
  6. For the profile photo on the About page, find src="images/profile-photo.svg" in about.html and replace it the same way.
